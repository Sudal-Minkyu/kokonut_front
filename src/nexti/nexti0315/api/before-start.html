시작하기 전에

코코넛에 오신 것을 환영합니다.

코코넛에서는 개발사들이 쉽게 결제를 연동할 수 있도록 모든 플랫폼별 SDK를 제공합니다. 개발환경, PG에 상관없이 쉬운 문법, 적은 개발 공수를 제공하고, 구매자가 결제 진행함에 있어 불편한 시나리오가 없도록 끊임없이 개선하고 있습니다. 또한 결제 오류나 개발사 실수로 인해 결제처리가 누락되지 않도록 진화된 SDK와 아키텍쳐를 제공하고 있습니다.
코코넛의 모든 SDK는 직접 제작하고 관리합니다. 따라서 요청주시는 의견이나 기술문의에 대해 24시간 이내 응답이 가능하며, 빠른 속도로 패치할 수 있습니다. 또한 확장되는 코코넛의 내부 기능에 대해 모든 SDK가 바로 반영됩니다.
코코넛의 여동 과정을 간추리면 아래와 같습니다.

클라이언트 개발

서버 개입없어도 클라이언트에서 결제 진행까지 가능합니다. 클라이언트 SDK는 아래 기능을 지원합니다.

PG 결제창 띄우기, 서버없이 결제 완료까지 모두 진행 가능

본인인증 창 띄우기, 서버없이 인증 완료까지 모두 진행 가능

서버 개발

클라이언트의 결제정보는 언제나 위변조 될 수 있기 때문에, 보안상 서버 구현이 필요합니다. 만약 서버리스 환경을 구축하고자 검토하신다면, 웹훅 통지를 수신할 람다 함수를 검토하시길 추천합니다. 또한 보안상 고정 IP를 설정해야 하는데, 서버리스 환경에서 고정 IP를 설정할 수 없을 경우, 모든 IP 허용 설정으로 대체할 수 있지만, 그 만큼 보안이 취약해짐으로 권고하지 않습니다.
