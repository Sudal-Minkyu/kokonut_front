REST API

이 문서는 REST API를 사용한 카카오 API 사용법과 카카오디벨로터스(kakao Developers, 이하 개발자 웹사이트) 설정에 대하여 안내합니다.

기능 소개

카카오 API는 대부분의 기능을 REST API로 제공합니다. 카카오톡 공유나 서비스 UI와 관련 있는 플러그인 성격의 일부 기능을 제외하고 모든 기능을 서비스 서버에서 REST API로 구현할 수 있습니다. 또한 카카오 SDK가 지원하지 않는 카카오 API는 REST API를 통해 사용 가능합니다.

REST API는 HTTP 요청을 보낼 수 있는 환경이라면 어디서든 이용 가능합니다. 클라이언트에서 서버로, 또는 서버에서 서버로 요청을 보낼 수 있습니다. 예를 들면 다음과 같은 환경에서 REST API를 사용할 수 있습니다.

모바일/PC 웹 환경

iOS, Android 등 모바일 환경

Java, Rudy, Python 등 다양한 환경의 웹 서버

REST API 요청과 응답 규격을 비롯한 자세한 정보는 REST API 레퍼런스에서 확인할 수 있습니다.

발링키로 결제 요청하기

REST API 테스트 도구

위 과정에서 빌링키를 발급받아 저장하셨다면, 원하는 이벤트 시점(ex. 일주일 뒤에, 택시에서 내렸을 때)에 결제 승인 요청할 수 있습니다. 이때 구매자가 별도로 결제창 UI를 통해 진행하지 않아도, 개발사의 서버에서 저장된 빌링키와, 결제금액, 상품명 등의 정보를 REST API를 통해 결제 요청하면, 요청이 유효 할 경우 최종적으로 승인이 됩니다. 결제결과(성공/실패)는 서버로 통지됩니다.

  • PG사 다날 이용 시 빌링키 결제 방화벽 IP 해제 요청

    • 다날을 이용하는 개발사의 경우 방화벽애 의해 빌링키 결제 요청이 실패할 수 있습니다. 이때 다날측에 223.130.82.4, 223.130.82.130, 223.130.82.131에 대한 공용 IP를 방화벽에서 해제해달라고 요청해주시면 됩니다. 그렇지 않을 경우, RETURNCODE : 11112, RETURNMSG : 접근 권한이 없습니다. 같이 리턴받거나, PG사 방화벽에 의해 요청 자체가 실패할 수 있습니다.

  • 요청 Parameters

    • billing_key

      필수string

      가맹점에서 부여한 주문 고유값

      필수string

      자동결제 금액

      tax_free

      card_quota
